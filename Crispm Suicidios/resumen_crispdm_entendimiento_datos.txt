CRISP-DM METHODOLOGY SUMMARY
Focus: Data Understanding (CRISP-DM workshop)

CRISP-DM is a standard methodology for data mining projects and consists of 6 phases:
1. Business understanding
2. Data understanding
3. Data preparation
4. Modeling
5. Evaluation
6. Deployment

PHASE 2: DATA UNDERSTANDING
Objective:
Understand data structure, content, and quality before building models.

Main activities:
1. Collect initial data
- Identify information sources.
- Verify temporal and population coverage.
- Define the scope of available variables.

2. Describe the data
- Number of records and variables.
- Data types (numeric, categorical, date, text).
- Data dictionary and field meaning.

3. Explore the data (EDA)
- Descriptive statistics (mean, median, dispersion, percentiles).
- Variable distributions and behavior.
- Relationships between variables (correlations, category cross-tabs).
- Pattern and trend identification.

4. Verify data quality
- Missing values.
- Duplicates.
- Format or coding inconsistencies.
- Outliers.
- Data entry errors.

5. Formulate findings and initial hypotheses
- Potential factors associated with the studied phenomenon.
- Key variables for later preparation and modeling phases.

DELIVERABLES FOR THIS PHASE
1. Data source and structure report.
2. Variable profiling.
3. Descriptive and visual exploration results.
4. List of data quality issues and treatment plan.
5. Initial conclusions to guide preparation and modeling.

CONCLUSION
The CRISP-DM data understanding phase reduces modeling errors, improves analytical quality, and supports evidence-based decisions. It is the technical foundation for reliable and useful downstream phases.
