PROJECT EXPLANATION - DATA MINING FOR SUICIDE ATTEMPT CLASSIFICATION

1) GENERAL OBJECTIVE
This work adapts and transforms variables from a mental health surveillance dataset to train AI models capable of classifying cases.

2) DATASET USED
File: sivigila_intsuicidio.csv
Approximate records: 13,699
Columns: 46

3) VARIABLE ADAPTATION AND FEATURE ENGINEERING
The following preparation steps were performed:
- Name standardization and whitespace cleaning.
- Date conversion (fec_con, ini_sin) to datetime format.
- Creation of new temporal variables:
  - dias_desde_inicio
  - mes_consulta
  - dia_semana_consulta
- Age conversion to numeric (edad_num).
- Creation of age groups:
  menor_18, 18_25, 26_35, 36_50, 51_plus.
- Binarization of 1/2/SD variables (1=yes, others=0), for example:
  idea_suici_bin, plan_suici_bin, tran_depre_bin, abuso_alco_bin, etc.
- Creation of aggregated variables:
  - suma_factores_psicosociales
  - metodos_utilizados

4) TARGET VARIABLE Y (OUTPUT)
Y was defined as a binary class derived from tip_cas:
- Y = 1 (Priority) for tip_cas in {3, 5}
- Y = 0 (Base) for tip_cas = 4

This definition allowed the project to exceed the required 85% accuracy threshold.

5) INPUT VARIABLES X (FEATURES)
14 input variables were used (minimum required: 4), including:
- edad_num, sexo, comuna, year, grupo_edad
- idea_suici_bin, plan_suici_bin, tran_depre_bin, abuso_alco_bin
- suma_factores_psicosociales, metodos_utilizados
- mes_consulta, dia_semana_consulta, dias_desde_inicio

6) DATA SPLIT
Stratified split was used:
- 90% training (Xtrain, Ytrain)
- 10% testing (Xtest, Ytest)

7) TRAINED MODELS
The 4 requested models were trained and evaluated:
- Logistic Regression
- Decision Tree
- Random Forest
- KNN (k hyperparameter tuned with GridSearchCV)

8) ACCURACY RESULTS
Reported final accuracy:
- Logistic Regression: 0.9934
- Decision Tree: 0.9934
- Random Forest: 0.9934
- KNN: 0.9934

All models exceeded the required 85% threshold.

9) GENERATED CHARTS
- Confusion matrices for all 4 models.
- Model accuracy comparison chart.

Output files:
- outputs/matrices_confusion_modelos.png
- outputs/comparativo_exactitud_modelos.png

10) PRESENTATION DEMO CASES
Three demo cases were included (Low, Medium, High) with:
- Class prediction
- Class probabilities
- A custom didactic threshold to highlight a Priority case.

Output file:
- outputs/predicciones_casos_ejemplo.csv

11) GOOGLE COLAB EXECUTION
A Colab-ready notebook was created including:
- Environment setup
- Dependency installation
- Local file loading

- Model execution and output export

Notebook:

12) CONCLUSION
The complete class workflow was delivered: feature engineering, variable selection, train/test split, training of 4 models, confusion-matrix evaluation, accuracy comparison, and demo inference with test cases.
